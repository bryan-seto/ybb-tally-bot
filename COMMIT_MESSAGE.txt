Fix balance calculation to correctly handle overpayments and underpayments

Problem:
- The previous calculation used Math.max(0, ...) which hid negative values
- When one person overpaid, the system didn't correctly calculate what the other person owes
- Example: If Bryan overpaid $128.67, the old logic would show bryanOwes=0 instead of calculating that HY owes Bryan

Solution:
- Calculate net amounts: paid - share (positive = overpaid, negative = underpaid)
- If Bryan overpaid and HY underpaid: HY owes (Bryan's overpayment + HY's underpayment)
- If HY overpaid and Bryan underpaid: Bryan owes (HY's overpayment + Bryan's underpayment)
- Properly handles all edge cases
